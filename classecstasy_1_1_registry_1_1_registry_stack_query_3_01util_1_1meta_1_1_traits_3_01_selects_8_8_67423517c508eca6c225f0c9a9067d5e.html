Detailed Description

template<typename... Selects, typename... Missings, typename Condition, typename... Cs>
class ecstasy::Registry::RegistryStackQuery< util::meta::Traits< Selects... >, util::meta::Traits< Missings... >, Condition, util::meta::Traits< Cs... > >

Registry query allocating everything on the stack (if allocation required).

This means longer compile time for faster runtime.

The goal is to return a single object containing the query object and the modifiers allocator. Thus the lifetime of the query and of the modifiers are bound on the stack.

This was quite difficult to achieve since the RegistryStackQuery needs the allocator and the query, but the query needs the allocator... The solution I found is using multiple inheritance to call a function before the query creation. In our case the function is the AllocatorWrapper constructor. This ensure a safe allocator initialization just before the query creation and without move operations (meaning memory copy and usually pointers invalidation).
